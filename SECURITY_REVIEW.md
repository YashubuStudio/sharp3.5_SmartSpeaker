# セキュリティ／危険性チェック結果（2026-04-06）

対象: Smart Speaker リポジトリ全体

## 重点的に確認した観点
- 外部通信先の安全性（平文HTTP、誤接続、資格情報埋め込み）
- ファイル操作の安全性（パストラバーサル、シンボリックリンク）
- ダウンロード処理の安全性（サイズ無制限、破損データ）
- ログへの機微情報出力
- 異常系での停止（panic等）

## 主な所見

### 1) エンドポイント誤設定による漏えいリスク（対策済み）
- `llm.endpoint` / `tts.endpoint` は設定値次第で外部ホストへ通信可能。
- これまでは非localhost時に警告ログのみで、`http://` のまま動作可能だった。
- **対策**: 設定バリデーションで以下を必須化。
  - URLとして妥当
  - スキームは `http/https` のみ
  - username/password の埋め込み禁止
  - localhost以外は `https` 必須

### 2) モデルダウンロード時の容量上限なし（対策済み）
- Whisperモデル取得時、実データサイズに対する上限がなかったため、
  想定外サイズでのディスク圧迫リスクがあった。
- **対策**: 既知モデルサイズの120%を上限にし、
  - `Content-Length` が上限超過なら即中断
  - 受信途中で上限超過しても即中断

### 3) 既に実装済みの良い対策（確認）
- RAGのknowledge読込で、シンボリックリンク除外・正規化パス検証あり。
- TTS応答の音声サイズ上限（50MB）あり。
- 非localhostエンドポイント利用時に警告ログあり。

## 残課題（今回未実装）
- モデルファイルの**ハッシュ検証**（改ざん検出）
- ログ出力の機微情報マスキング（`You said: ...` 等）
- `panic!` を使った分岐（主にテスト用途だが、将来的には `Result` 統一が望ましい）

## 追加推奨
1. リリース時に `cargo audit` をCIで実行
2. 外部通信先をallowlist化（必要ならCIDRも）
3. モデルのSHA-256固定値検証を導入
